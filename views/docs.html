Posts

All the paths starts with /posts --> Ex. for end point /add-post the path becomes /posts/add-post

Authentication token -which is valid for only 1 hour- must be included in the header of each request of the movies

End Points Request Method message Notes
/ GET All the Posts pagination is always on when you get all posts or search for posts title
query params (title) are for search for Ex.
posts?title=h --> search by posts title containing letter h (cas insenitive)
/:postId GET Requested Post Each Post has its own Comments,number Of Comments , number Of Reactions
/add-post POST Post Created req body Ex.:
{"title":"How to be Fit","content":"By Sports"}
ofcourse
/edit-post/:postId PATCH Post Updated ---
/:postId DELETE Post Removed ---
/add-comment/:postId POST Comment Added req body Ex.:
{"content":"Great Post"}
/:postId/edit-comment/:commentId PATCH Comment Updated ---
/:postId/remove-comment/:commentId DELETE Comment Removed ---

Authentication

All the pathes starts with /auth --> Ex. for end point /login the path becomes /auth/login

End Points Request Method message Notes
/signUp POST User Signed up Successfully req body Ex.:
{"name":"...","email":"test@test.com","password":'123456',"confirmedPassword":'123456'}
the email field must contain a valid email address
If the same email found in the DB an error message occured ('Email Already Exists')
password with min length = 5 and max length = 12
name field cannot be empty
ofcourse password and confirmed password must be equal
/login POST User Signed In Successfully {"email":"test@test.com","password":'123456'}
If the email is not found in the DB A error message occured ('No User with this email')
password with min length = 5 and max length = 12
